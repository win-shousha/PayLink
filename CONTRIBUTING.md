# Contributing to PayLink

Merci de votre intérêt pour contribuer à PayLink! Ce document fournit des lignes directrices et des instructions pour contribuer au projet.

## Code de Conduite

Tous les contributeurs sont tenus de respecter notre code de conduite. Veuillez consulter CODE_OF_CONDUCT.md pour plus de détails.

## Comment contribuer

### Signaler des bugs

1. Vérifiez que le bug n'a pas déjà été rapporté
2. Créez une nouvelle issue avec le titre clair
3. Décrivez exactement comment reproduire le bug
4. Fournissez des exemples spécifiques
5. Décrivez le comportement observé et ce que vous vous attendiez

### Suggérer des améliorations

1. Utilisez un titre clair et descriptif
2. Fournissez une description détaillée de la suggestion
3. Listez les exemples de façon à ce que la suggestion soit claire
4. Décrivez le comportement actuel et le comportement attendu

### Pull Requests

1. Forkez le dépôt
2. Créez une branche (`git checkout -b feature/AmazingFeature`)
3. Committez vos changements (`git commit -m 'Add some AmazingFeature'`)
4. Poussez vers la branche (`git push origin feature/AmazingFeature`)
5. Ouvrez une Pull Request

## Normes de Style

### JavaScript/TypeScript

- Utilisez ESLint pour la vérification de style
- Indentation de 2 espaces
- Utilisez des const par défaut, let si nécessaire
- Nommage camelCase pour les variables et fonctions
- Nommage PascalCase pour les classes et composants

### Vue Components

- Utilisez la composition API
- Organisez le template, script, puis les styles
- Utilisez des noms de composants en PascalCase avec deux mots

## Process de Review

1. Au minimum une approbation requise
2. Les tests doivent passer
3. Pas de conflits de fusion
4. La couverture de code ne doit pas diminuer

## Questions?

N'hésitez pas à contacter l'équipe ou ouvrir une issue!
